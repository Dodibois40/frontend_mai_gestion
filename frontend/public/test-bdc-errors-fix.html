Test - Corrections erreurs BDC

Validation des corrections des erreurs 400 (Bad Request)

Corrections apportées

Frontend : Correction de l'appel receptionnerBdc(bdcId) → receptionnerBdc(bdcId, new Date())
Backend : Ajout des décorateurs de validation au ReceptionBdcDto
Backend : Import des décorateurs @ApiProperty, @IsDate, @Type
PDF : Correction de l'erreur Cannot read properties of null (reading 'fileName')

Problèmes identifiés avant correction

Erreur 400 - Réception BDC
❌ /api/bdc/{id}/receptionner
Cause : AffaireBdcSectionReal.jsx appelait receptionnerBdc(bdcId) sans passer la date
Erreur 400 - Suppression BDC
❌ DELETE /api/bdc/{id}
Cause : Validation du ReceptionBdcDto sans décorateurs appropriés
Erreur PDF Upload
❌ TypeError: Cannot read properties of null (reading 'fileName')
Cause : handlePdfUploadSuccess ne gérait pas le cas result === null

Tests à effectuer

1 Créer un nouveau BDC et tenter de le réceptionner
2 Uploader un PDF sur un BDC puis le supprimer
3 Valider un BDC puis tenter de le supprimer (avec mot de passe)
4 Vérifier que les dates de livraison s'affichent correctement

Détails techniques des corrections

1. Frontend - AffaireBdcSectionReal.jsx

❌ Avant :
await receptionnerBdc(bdcId);
✅ Après :
await receptionnerBdc(bdcId, new Date());

2. Backend - ReceptionBdcDto

❌ Avant :
class ReceptionBdcDto {
  dateReception: Date;
}
✅ Après :
class ReceptionBdcDto {
  @ApiProperty({
    description: 'Date de réception du bon de commande',
    example: '2024-03-15T10:30:00Z',
  })
  @IsDate()
  @Type(() => Date)
  dateReception: Date;
}

3. Frontend - handlePdfUploadSuccess

✅ Correction :
if (result === null) {
  // Gestion spécifique de la suppression
  const updateData = {
    nomFichier: null,
    tailleFichier: null,
    // ...
  };
  // Continuer le traitement
}
